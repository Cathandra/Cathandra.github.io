Less知识点复习

Posted on 2019-04-21 In Front-End Views:

Less复习。
Less是一种CSS预处理语言，规避了CSS的逻辑性短板。个人认为，Less主要是对CSS添加了变量、混合与函数三个板块的功能，藉此简化开发工作。
那下面我就又要开始罗列知识点了。

变量

Less中定义的变量使用@开头，引用时直接使用@variable_name。

值变量

与其称之为变量，不如把变量理解为一个常量，因为其值本身就是不可以重新定义的。
这样，在编写样式文件时，对于常用变量，如主色调、固定的长度等等，可以直接进行通过变量名引用，不必重新写出值。

其他还有属性变量、选择器变量等。为了上下思路连贯，这里就不多写了。

变量运算

变量虽然不可修改，但是less支持对变量进行运算。在加减法中，单位自动与第一个变量统一。

混合

混合实质上可以理解为对方法的使用，所以归在下一节中。

方法

方法直接可以和声明的集合一样定义，但是为了作区分，最好定义时在方法后写 ()。

方法可以设置默认参数，然后在调用时直接使用或修改。

@arguments——全部参数

@arguments和很多语言中一样，指代的是全部参数。

匹配模式

可以和多态概念一样理解。

extend关键字——继承

使用extend可以继承所匹配声明中的全部样式。

混合模式

混合模式指，在需要使用该方法的元素样式中直接引用该方法。

&关键字——嵌套模式

使用&代表上一层选择器的名字。
嵌套与继承、与混合的区别在于，继承指继承所匹配声明中的全部样式、混合是引用了匹配样式，嵌套只是代表上一层选择器的名字。

@media——媒体查询

less的媒体查询可以直接在元素内编写，每一个元素都会编译出自己 @media 声明。

!important关键字

!important关键字指对于重复规则强制使用。在less中直接在方法名后加关键字即可。

其他

虽然不属于语言特性但好像还是应该提一下。

在Node.js环境中使用Less

编译后，.less文件的内容会以符合css语法的形式出现在同名的.css文件中。

注释和避免编译

/* */是CSS原生注释，会被编译在 CSS 文件中；
// 是Less提供的注释，不会被编译在 CSS 文件中。
